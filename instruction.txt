Предложения по дополнению инструкции по интеграции:

1. Дополнение к пункту "Получение токена доступа"
 - так как было необходимо вывести информацию о пользователе, можно добавить раздел с полями,
   которые содержатся в полученном токене, которые необходимы для получения дальнейшей информации
 - поле "sub" - содержит идентификатор авторизованного пользователя
2. Получение информации о пользователе
 - Чтобы получить информацию о пользователе, используйте метод Get user public profile
  (GET /users/{user_id}/public)
   с параметром user_id=sub, полученном в методе Generate JWT
 - Также в заголовок запроса "Authorization" необходимо передать Generate JWT
